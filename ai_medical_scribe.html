Welcome

AI Medical Scribe runs locally in your browser

This app is a browser-based medical scribe prototype for transcription, SOAP summaries, and draft document generation. Session data stays on this computer, with no backend controlled by this project. The project is open source on GitHub and is built by Findon Software.

Browser capability check

This app works best in recent Chrome builds with built-in AI enabled.

If browser AI is not ready yet, you can still use manual notes, session history, and local editing while Chrome downloads or enables the required model support.

Setup help

Enable Chrome built-in AI features

This prototype is designed for recent Chrome builds, usually served on localhost. For the best results, use Chrome Canary or a recent Chrome build with built-in AI flags enabled.

  1. Serve the app on http://localhost rather than opening the file directly.
  2. Open chrome://flags.
  3. Enable chrome://flags/#optimization-guide-on-device-model.
  4. Enable the Prompt API flag available in your build, such as chrome://flags/#prompt-api-for-gemini-nano or chrome://flags/#prompt-api-for-gemini-nano-multimodal-input.
  5. Relaunch Chrome and allow the on-device model to download if prompted.

You can inspect readiness in DevTools with LanguageModel.availability(...), and review chrome://on-device-internals or chrome://components if the model never becomes ready.

The README in this project contains the same setup notes in more detail.

Behaviour notes

This front-end uses the browser's speech recognition engine when available. The recognition provider is isolated in JavaScript so it can be replaced with another provider later.

Only one session is treated as active at a time. All session data remains in-browser with no backend calls.

On-device AI summary

Uses Gemini Nano through the browser when available. No API key is required.
